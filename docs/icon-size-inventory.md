# Icon size inventory

Exact location and effective size of every icon glyph rendered on the site.
Generated by auditing `src/components/**/*.astro` against `src/styles/global.css`.

Legend (read before the tables):

- **Size prop** — the `size` value written on the `<Icon>` tag (`"sm"`=16, `"md"`=20, `"lg"`=24, or a raw `{N}` px number).
- **Slot** — the wrapping element's class. A slot of `.icon-tile*` or `.theme-card__icon` triggers a CSS size override.
- **Governed by** — what actually sets the rendered pixel size:
  - `prop` = the `size` prop wins (standalone icon, no slot override).
  - `--mark-glyph` = `global.css:772-775` forces `width/height: var(--mark-glyph)` on any `svg`/`.mark-emblem` inside `.icon-tile`/`.theme-card__icon`; the `size` prop is ignored for dimensions.
  - `--vision-hub-glyph` = `VisionHub.astro:145-168` forces the fluid hub-glyph size.
- **Rendered px** — the true on-screen size after CSS.
- **Dead prop?** — `YES` when the icon is in a slot, so the written `size` value has no effect (and may not even match the rendered size).

## Key findings

- Only **9 of 15** direct icons are controlled by their `size` prop; the other 6 are inside slots where CSS `--mark-glyph` / `--vision-hub-glyph` wins.
- **The two misleading dead props have been removed** (both now render exactly as before — CSS already governed them):
  - `CompetitionCard.astro:109` — dropped `size="lg"` (claimed 24; renders **22** via `--mark-glyph`).
  - `VisionHub.astro:57` — dropped `size="md"` (claimed 20; renders the **fluid hub glyph** via `--vision-hub-glyph`).
  - Both now omit `size`, matching the in-slot convention (`RecogTile.astro:24`).
- **The redundant in-slot props have been removed** — `Education.astro:113` & `:152` (`{22}`) and `XpProjectCard.astro:29` (`{18}`) matched their slot's `--mark-glyph`, so dropping them changed nothing visually; the slots are now uniformly size-prop-free.
- **The only genuine non-token standalone size** is `Contact.astro:62` `size={18}` (booking-call glyph) — 18px has no `--icon-*` token and is not in a slot.
- `--icon-xs` (14) and `--icon-xl` (32) tokens exist but are unused by any `<Icon>`.
