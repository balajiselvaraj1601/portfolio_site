# Zero-Impact Refactor — 2026-07-06

Seven-phase internal refactor on `main`, released as v1.2.0. Baseline: tag
`refactor-baseline` (`e87958d`, the P0 harness commit itself). Every phase was
gated on `npm run verify` plus `scripts/dist-diff.mjs` against the baseline
build; component-decomposition phases additionally passed dual-theme screenshot
gates. The site output is byte-identical throughout except the enumerated
exception classes below — the only intentional visual change is the Contact
copy/spacing edit adopted as its own commit (`5b63ab9`).

## Phases

### P0 — dist-diff harness

- **What:** `scripts/dist-diff.mjs` compares two `dist/` builds after
  normalizing content hashes, `astro-cid` scoping attributes, and same-stem
  bundle collisions. Extended mid-refactor (`163ed98`) with a `--loose` mode
  for component-split phases: symmetric compare-time transforms collapse
  inter-tag whitespace runs (2+) to one space and flatten CSS media-aware into
  a rule multiset. Strict mode stays the default.
- **Why:** prove byte-equivalent output before touching anything; give P4's
  component splits a gate that ignores only provably render-inert deltas.
- **Gate:** self-hosting — the harness commit is the baseline tag.
- **Commits:** `e87958d` (harness, tagged `refactor-baseline`), `163ed98`
  (`--loose` mode).

## Dist-diff exception classes

Every non-zero dist-diff over the whole refactor falls into exactly one of
three enumerated classes; nothing else changed in `dist/`:

1. **P4 render-inert whitespace** — single inter-tag spaces dropped at new
   component boundaries, all in block / absolutely-positioned / SVG contexts
   where inter-tag whitespace does not render (e.g. 18 normalized diff lines
   in the Vision program-grid region; the SpeakingCard media anchor). Absorbed
   by `--loose` mode's symmetric whitespace collapse.
2. **Contact edit** — the intentional copy/spacing change adopted as
   `5b63ab9`; excluded from refactor gating by committing it on its own.
3. **P5 deletions** — the 79 enumerated dead CSS lines and the pruned
   unrendered content fields; deletions only, each verified zero-consumer.

## Decisions log

- **Renames deferred.** File/class renames (e.g. legacy `recog-*` naming on
  live classes) were considered and deferred: the name-coupling inventory
  showed each rename fans out across agent docs, skills, routing CSV, tests,
  and frozen audit history — which must stay untouched — so a rename buys no
  output change while guaranteeing either a large doc sweep or permanent
  doc/code drift.
- **Kaggle `realWorldImpact`-class keys pruned, not ratified.**
  `realWorldImpact`/`teamMembers`/`conferenceTrack` were never in the schema
  (Zod strips unknown keys, so they never rendered); they were removed from
  the JSON rather than promoted into the schema.
- **Contact seam rejected.** Contact.astro was assessed for P4 decomposition
  and deliberately left intact: grid-coupled sizing rules span its only viable
  seam, so any split would separate rules that must co-evolve.
